# 브로커 다중화 및 레플리카 증가 작업계획

## 목표

브로커 1개 단일 구조에서 브로커 3개 클러스터 구조로 전환하고, 레플리카를 3개로 설정하여 고가용성을 확보한다.

## 작업 범위

- 1단계: 브로커 3개로 확장
- 2단계: 레플리카 3개로 설정

---

## 세부 작업 목록

### 1단계: docker-compose.yml 브로커 서비스 분리

- [x] 기존 `kafka` 서비스를 `kafka1`, `kafka2`, `kafka3`으로 분리
- [x] 각 브로커에 고유한 `KAFKA_BROKER_ID` 설정 (1, 2, 3)
- [x] 각 브로커의 호스트 포트 분리 (9092, 9093, 9094)

### 2단계: 레플리카 3개로 설정

- [x] `kafka1`의 `KAFKA_CREATE_TOPICS`를 `system-metrics:1:3`으로 변경 (파티션1, 레플리카3)

### 3단계: kafka-ui 설정 업데이트

- [x] `KAFKA_CLUSTERS_0_BOOTSTRAPSERVERS`에 3개 브로커 주소 모두 등록 (`kafka1:9092,kafka2:9092,kafka3:9092`)
- [x] `depends_on`을 `kafka1`, `kafka2`, `kafka3`으로 업데이트

### 4단계: producer 설정 업데이트

- [x] `producer/main.py`의 `BOOTSTRAP_SERVERS`를 3개 브로커로 업데이트
- [x] `docker-compose.yml`의 producer `depends_on`을 `kafka1`, `kafka2`, `kafka3`으로 업데이트

### 5단계: consumer 설정 업데이트

- [x] `consumer/main.py`의 `BOOTSTRAP_SERVERS`를 3개 브로커로 업데이트
- [x] `docker-compose.yml`의 consumer `depends_on`을 `kafka1`, `kafka2`, `kafka3`으로 업데이트

---

## 변경 대상 파일

| 파일 | 변경 내용 |
|------|-----------|
| `docker-compose.yml` | 브로커 3개로 분리, 레플리카 설정 변경 |
| `producer/` | bootstrap.servers 업데이트 |
| `consumer/` | bootstrap.servers 업데이트 |

---

## 완료 기준

- 브로커 3개가 정상 기동되고 클러스터로 묶임
- 브로커 하나를 강제 종료해도 프로듀서/컨슈머가 정상 동작
- kafka-ui에서 3개 브로커와 레플리카 3개 확인 가능
