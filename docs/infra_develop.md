# 인프라 개선 계획 1 - 안정성 개선

## 현재 구조의 문제

```
브로커 1개 + 파티션 1개 + 레플리카 1개
→ 브로커가 죽으면 전체 서비스 중단
```

## 개선 순서

### 1단계: 브로커 3개로 확장

```
브로커1, 브로커2, 브로커3
```

가장 핵심. 브로커 하나가 죽어도 서비스 유지.

### 2단계: 레플리카 3개로 설정

```
system-metrics:1:3  # 파티션1, 레플리카3
```

브로커가 3개가 되면 의미있어짐. 데이터 유실 방지.

### 3단계: 파티션 늘리기 (선택)

처리량이 문제가 될 때 고려. 안정성보다는 성능 개선 목적.

### 4단계: Zookeeper도 다중화

```
Zookeeper 1개 → 3개
```

Zookeeper가 단일 장애점(SPOF)이라 함께 다중화 필요.

## 우선순위 요약

브로커 다중화 → 레플리카 증가 → Zookeeper 다중화
