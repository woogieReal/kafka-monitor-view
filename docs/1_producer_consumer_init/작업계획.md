## Producer / Consumer 초기 구현 작업계획

### 1. 언어 및 라이브러리

*   **언어**: Python
*   **OS 지표 수집**: `psutil`
*   **Kafka 클라이언트**: `kafka-python`
*   **선택 이유**: 코드가 짧고 직관적이며, 로컬 테스트 목적에 적합함.

### 2. 디렉토리 구조

```
producer/
├── Dockerfile
├── requirements.txt
└── main.py

consumer/
├── Dockerfile
├── requirements.txt
└── main.py
```

### 3. Metric Producer 구현 계획

**파일**: `producer/main.py`

#### Step 1. 환경 설정 (`requirements.txt`, `Dockerfile`)

*   `requirements.txt`에 의존 라이브러리 명시
    ```
    psutil
    kafka-python
    ```
*   `Dockerfile` 작성
    *   베이스 이미지: `python:3.11-slim`
    *   `requirements.txt` 복사 후 `pip install` 실행
    *   `main.py` 복사 후 `CMD ["python", "main.py"]`로 실행

#### Step 2. OS 지표 수집

*   `psutil.cpu_percent(interval=1)`로 CPU 사용률(%) 수집
    *   `interval=1`: 1초간 측정 후 반환 → 2초 루프 주기와 합산되어 실질적으로 약 3초 주기가 되므로, `interval=None`으로 설정하고 루프 마지막에 `time.sleep(2)` 처리
*   `psutil.virtual_memory().available`로 남은 메모리(bytes) 수집
*   `node_id`는 `socket.gethostname()`으로 컨테이너명 자동 취득

#### Step 3. JSON 메시지 직렬화

*   `app.md` 스키마 기준으로 dict 구성 후 `json.dumps()` 직렬화
    ```json
    {
      "timestamp": "2026-05-01T16:46:00Z",
      "node_id": "local-mac-mini",
      "metrics": {
        "cpu_usage": 1.25,
        "mem_free_bytes": 1073741824
      }
    }
    ```
*   `timestamp`는 `datetime.utcnow().isoformat() + "Z"` 형식으로 생성
*   직렬화된 문자열을 `utf-8`로 인코딩하여 Kafka 메시지 value로 사용

#### Step 4. Kafka 발행

*   `KafkaProducer` 초기화 시 `bootstrap_servers=["kafka:9092"]` 설정
*   `producer.send("system-metrics", value=message_bytes)` 호출
*   `producer.flush()`로 전송 완료 보장
*   Kafka 연결 실패 시(`NoBrokersAvailable`) 재시도할 수 있도록 초기화를 루프 외부에서 처리하고, 예외 발생 시 로그 출력 후 프로세스 종료

#### Step 5. 반복 루프

*   `while True` 루프 안에서 Step 2~4 반복
*   각 발행 후 `time.sleep(1)` 대기
*   루프마다 발행 성공 여부를 `print()`로 터미널에 출력하여 동작 확인 가능하게 함

### 4. Alert Consumer 구현 계획

**파일**: `consumer/main.py`

#### Step 1. 환경 설정 (`requirements.txt`, `Dockerfile`)

*   `requirements.txt`에 의존 라이브러리 명시
    ```
    kafka-python
    ```
*   `Dockerfile` 작성
    *   베이스 이미지: `python:3.11-slim`
    *   `requirements.txt` 복사 후 `pip install` 실행
    *   `main.py` 복사 후 `ENV PYTHONUNBUFFERED=1`, `CMD ["python", "main.py"]`로 실행

#### Step 2. 토픽 구독 및 메시지 파싱

*   `KafkaConsumer` 초기화 시 `bootstrap_servers=["kafka:9092"]` 설정
*   `auto_offset_reset="latest"` — Consumer 시작 시점 이후의 메시지만 수신
*   `value_deserializer`로 수신 즉시 JSON 파싱 (`json.loads` + `utf-8` 디코딩)
*   Kafka 연결 실패 시(`NoBrokersAvailable`) 5초 간격으로 재시도
*   `for message in consumer` 루프로 메시지 상시 대기

#### Step 3. 임계치 판단 및 경고 출력

*   파싱된 메시지에서 `metrics.cpu_usage` 추출
*   `cpu_usage > 1.0` 초과 시 터미널에 경고 출력
*   정상 범위일 경우 수신 로그만 출력

#### Step 4. `docker-compose.yml` 등록

*   consumer 서비스 추가 (`build: ./consumer`)
*   `restart: on-failure` 설정 (Kafka보다 먼저 뜰 경우 자동 재시작)
*   `depends_on: kafka` 설정

### 5. 공통 사항

*   브로커 연결 주소: `kafka:9092` (Docker 네트워크 내부 통신)
*   두 서비스 모두 `docker-compose.yml`에 추가하여 컨테이너로 실행
