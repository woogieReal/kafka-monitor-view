## [2026-05-01] Producer / Consumer 초기 구현

### 작업 내용
- Python 기반 Metric Producer 구현 (`producer/main.py`)
  - `psutil`로 CPU 사용률, 가용 메모리 수집
  - `socket.gethostname()`으로 node_id 자동 취득
  - JSON 직렬화 후 Kafka `system-metrics` 토픽에 1초 주기로 발행
- Alert Consumer 구현 (`consumer/main.py`)
  - `system-metrics` 토픽 구독, `auto_offset_reset="latest"` 설정
  - `cpu_usage > 1.0` 초과 시 경고 출력, 정상 범위는 수신 로그만 출력
- Producer / Consumer용 `Dockerfile` 및 `requirements.txt` 작성
- `docker-compose.yml`에 producer, consumer 서비스 추가

### 주요 결정 사항
- `psutil.cpu_percent(interval=None)` 사용: `interval=1`을 쓰면 1초 블로킹이 발생해 루프 주기가 약 3초로 늘어남. `interval=None`으로 즉시 반환받고 루프 끝에 `time.sleep(1)`로 주기를 통일
- Kafka 연결 실패(`NoBrokersAvailable`) 시 종료 대신 5초 간격 재시도: 컨테이너 기동 순서 불일치를 `restart: on-failure`와 함께 방어
- `auto_offset_reset="latest"`: Consumer 기동 시점 이전 누적 메시지를 처리하지 않고 실시간 메시지만 수신
- `datetime.now(timezone.utc)` 사용: deprecated된 `datetime.utcnow()` 대신 timezone-aware 방식으로 UTC 타임스탬프 생성
