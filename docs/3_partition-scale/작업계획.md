# 파티션 증가 작업계획

## 목표

`system-metrics` 토픽의 파티션을 1개에서 3개로 늘려 처리량을 개선한다.

## 작업 범위

- 파티션 수 증가: 1 → 3
- producer/consumer 동작 확인

---

## 세부 작업 목록

### 1단계: 토픽 파티션 수 변경

- [x] `docker-compose.yml`의 `KAFKA_CREATE_TOPICS`를 `system-metrics:3:3`으로 변경 (파티션3, 레플리카3)

### 2단계: consumer 파티션 할당 확인

- [x] consumer에 `group_id` 추가하여 파티션 할당 활성화
- [x] 메시지 로그에 `partition` 번호 출력하여 분산 여부 확인 가능하도록 수정

### 3단계: 동작 검증

- [x] `kafka-topics.sh --describe`로 파티션 3개, 각 파티션 리더가 kafka1/2/3에 분산 확인
- [x] consumer 로그에서 `partition=` 필드로 수신 파티션 확인 가능
- 단일 producer + no key → sticky partitioning으로 모든 메시지가 partition=0에 집중되는 것은 정상 동작
- 실제 분산은 여러 producer 인스턴스 또는 메시지 key 지정 시 발생

---

## 변경 대상 파일

| 파일 | 변경 내용 |
|------|-----------|
| `docker-compose.yml` | `system-metrics:1:3` → `system-metrics:3:3` |

---

## 완료 기준

- kafka-ui에서 `system-metrics` 토픽의 파티션이 3개로 표시됨
- producer 메시지가 여러 파티션에 분산되어 쌓임
- consumer가 정상적으로 메시지를 소비함
